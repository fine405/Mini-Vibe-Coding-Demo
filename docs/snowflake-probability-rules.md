# 雪花收集概率规则

> 版本：v1.0  
> 用途：雪花掉落、收藏成就、概率展示与研发实现  
> 基准：每批 10 片雪花；每片独立抽取；允许重复；无保底

## 1. 规则摘要

每次触发下雪事件时，系统会生成若干片雪花。每一片雪花都从同一个固定权重池中独立抽取一种类型。

- 每片雪花必定抽中 16 种类型之一，不存在“空结果”。
- 同一批内允许出现多个相同类型。
- 不同批次之间互不影响。
- 首次抽中某个稳定 ID 时，解锁对应收藏。
- 重复抽中不会提高后续稀有类型的概率。
- 不设置保底、重复保护、动态升权或必得机制。
- 概率只由固定权重决定，与玩家历史、已收藏数量、连续未获得次数无关。
- 当前概率分析以每批 `M = 10` 为基准；若以后修改批量大小，单片概率保持不变，但每批出现率和收集节奏需要重新计算。

这套规则刻意保留长尾：稀有雪花可能很快出现，也可能在很长时间内一直不出现。

## 2. 核心抽取模型

### 2.1 单片抽取

总权重固定为：

```text
TOTAL_WEIGHT = 100000
```

某种雪花 `i` 的单片出现概率为：

```text
p_i = weight_i / TOTAL_WEIGHT
```

每次单片抽取都必须返回且只返回一种雪花。16 种雪花的权重总和必须等于 `100000`，对应单片概率总和 `100%`。

### 2.2 批量掉落

若一批生成 `M` 片雪花，则连续执行 `M` 次相互独立的单片抽取：

```text
batch = [draw(), draw(), ..., draw()]  // 共 M 次
```

因此：

- 一批一定生成 `M` 片雪花。
- 一批可能包含多个不同类型。
- 同一类型也可能在一批中重复多次。
- 各种雪花的“每批至少出现一次”概率不需要加总为 `100%`。

### 2.3 收藏解锁

收藏状态只记录某个稳定 ID 是否至少出现过一次：

```text
首次出现：locked -> unlocked
重复出现：unlocked -> unlocked
```

收藏状态不得反向修改概率池，也不得改变任何权重。

## 5. 批次出现率与收集难度

### 5.1 一批至少出现一次

某种雪花单片概率为 `p_i`，一批生成 `M` 片时，该类型在整批中至少出现一次的概率为：

```text
q_i(M) = 1 - (1 - p_i)^M
```

### 5.2 首次出现所需批数

以“批”为单位连续独立抽取时，首次出现服从几何分布。首次出现所需批数的期望为：

```text
E_i(M) = 1 / q_i(M)
```

在 `n` 批以内至少出现一次的概率为：

```text
F_i(n, M) = 1 - (1 - p_i)^(n × M)
```

达到目标把握度 `c` 所需的最少批数为：

```text
n_c = ceil( ln(1 - c) / (M × ln(1 - p_i)) )
```

## 6. 长尾效果说明

以最稀有的 `snow_geometric` 为例：

- 单片概率为 `0.01%`，即平均每 `10,000` 片出现一次。
- 每批 10 片时，每批至少出现一次的概率约为 `0.1%`。
- 首次出现的期望约为 `1,000.5` 批。
- 经过约 `2,303` 批后，仍有约 `10%` 的玩家没有获得。
- 即使超过 `2,303` 批，也仍然不存在下一批必出的保证。

这种长尾正是收藏玩法的难度来源。产品文案不应把期望值描述为“最多”“必定”或“保底”。

## 7. 全图鉴完成节奏

在每批固定 10 片、所有单片独立抽取、无保底的前提下，当前权重对应的全 16 种收藏节奏约为：

| 指标 | 完成全图鉴所需批数 |
|---|---:|
| 平均完成批数 | 约 1,196 批 |
| 50% 玩家完成 | 约 933 批 |
| 90% 玩家完成 | 约 2,384 批 |
| 95% 玩家完成 | 约 3,041 批 |
| 99% 玩家完成 | 约 4,615 批 |

这些值是概率分布结果，不是进度承诺。因为没有保底，理论上不存在有限的“最晚完成批数”。全图鉴时长主要受 `snow_leaf_plate` 和 `snow_geometric` 两个传说类型影响。

## 8. 同批重复与数量分布

某种雪花在一批 `M` 片中恰好出现 `k` 次的概率服从二项分布：

```text
P(X_i = k) = C(M, k) × p_i^k × (1 - p_i)^(M - k)
```

该类型每批出现数量的期望为：

```text
E[X_i] = M × p_i
```

因此，同批重复不是异常情况，不应在抽取后去重，也不应因为已经抽中过某个类型而重抽。

## 9. 批量大小变更规则

当前所有难度表以固定 `M = 10` 计算。如果视觉层需要调整每批数量：

1. 单片权重表保持不变。
2. 每批实际生成多少片，就执行多少次独立抽取。
3. 固定批量 `M` 可直接使用 `q_i(M) = 1 - (1 - p_i)^M`。
4. 若批量大小本身是随机变量，则每批出现率应计算为：

```text
q_i = 1 - E[(1 - p_i)^M]
```

仅仅保持“平均每批 10 片”不代表收集节奏与固定 10 片完全相同。正式采用随机批量前，应按实际批量分布重新计算本文件中的批次指标。

## 10. 推荐实现

### 10.1 权重抽取伪代码

```text
function drawSnowflake(): Snowflake {
  roll = randomInteger(1, 100000)
  cumulative = 0

  for item in SNOWFLAKE_POOL:
    cumulative += item.weight
    if roll <= cumulative:
      return item

  unreachable
}

function dropBatch(count = 10): Snowflake[] {
  results = []
  repeat count times:
    results.push(drawSnowflake())
  return results
}
```

### 10.2 实现约束

- 使用整数权重，避免浮点边界误差。
- 随机整数区间必须完整覆盖 `1...100000`，且每个整数等概率。
- 概率池顺序可以调整，但 ID 与权重映射不得改变。
- 抽取完成后不得去重、补抽或替换结果。
- 收藏解锁在抽取结果确定后执行，且不得影响后续抽取。
- 动画时序、位置、大小、旋转和透明度只影响表现，不得影响类型概率。
- 若需要可复现测试，应由随机数生成器注入固定种子，而不是修改权重。

## 11. 前端展示口径

- 收藏页中每种雪花的主概率建议展示“单片概率”。
- 如需展示“本批出现率”，必须注明对应的批量大小，例如“每批 10 片时约 0.1%”。
- 不要把不同类型的“每批至少一次”概率相加，因为同一批可以同时出现多种类型。
- 稀有度标签只表示展示分层，不代表先抽稀有度、再抽具体类型。
- 已解锁与未解锁状态仅影响 UI，不影响概率。
- 对外文案应明确“独立概率、允许重复、无保底”。

## 12. 数据与验收检查

上线前至少完成以下检查：

- [ ] 16 个稳定 ID 全部唯一且非空。
- [ ] 所有权重均为正整数。
- [ ] 权重总和严格等于 `100000`。
- [ ] 单片抽取永远返回且只返回一种雪花。
- [ ] 一批 10 片时严格执行 10 次独立抽取。
- [ ] 同批重复不会被去重或替换。
- [ ] 收藏状态、玩家历史和连续未获得次数不会改变权重。
- [ ] 不存在保底、重复保护或隐藏升权逻辑。
- [ ] 大规模模拟结果与配置概率在合理抽样误差内一致。
- [ ] 稀有类型使用足够大的模拟样本验证，避免因样本量过小得出错误结论。
- [ ] 概率调整时同步更新本文件版本、配置表和难度表。

## 13. 变更原则

概率配置发布后如需调整：

1. 不修改已经发布的稳定 ID。
2. 先调整整数权重，并保证总和仍为 `100000`。
3. 重新计算每批出现率、首次出现期望、90% 获得批数和全图鉴节奏。
4. 在版本记录中说明调整原因和预期影响。
5. 不通过未公开的玩家分组或动态权重实现差异化概率。
